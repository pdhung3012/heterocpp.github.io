Heterogeneous Graph Neural Networks for Unifying Textual and Programming Artifacts

  • Introduction
  • Approach

Along with code optimization, research on textual artifacts has been interesting recently. More and more research focuses on textual artifacts like natural language requirements and code documentation, which describe the properties of the software projects, commits, or documentation at the method declaration level. In this work, we focus on non-documentation code comment, which is the textual artifact that appeared inside the body of the method declaration. Compared to other types of documentation comments, non-documentation comments are used to describe properties of code at smaller scopes as statements or blocks of statements. Moreover, along with the comments, information about the surrounding code of code comment is another resource that can help in mining the properties of the implementation of code comment.

In this work, we analyze the abilities of Heterogeneous Graph Neural Networks to take advantage of unified representation from graphs of both code comments and their surrounding context. The intuition of our work is to analyze the performance of the combination of Abstract Syntax Tree in programming language and statistical parse tree in natural language for program analysis. From the dataset SPoC of C/C++ code snippets with code comments about the hidden implementation of a statement, our first prediction task is to find the type of statement of the unseen implementation using code comments. In our second prediction task, we further extend your HGT model to support Neural Machine Translation to automatically identify the compiler errors without running the C/C++ compiler. Our experiment shows that our classification models achieve 78.44% and 74.53% accuracy for two prediction tasks. We significantly outperform the traditional homogeneous GNN models, random forest models, and deep learning LSTM models.
